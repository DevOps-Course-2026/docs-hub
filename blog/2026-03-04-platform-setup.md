## What This Is

The `docs-hub` is the central documentation engine for my DevOps course organization on GitHub. The idea: every repo in the org has a `/docs` folder, and it auto-syncs here. One rendered portal, many sources of truth.

Stack: Docusaurus 3.9 on Node 22, deployed to GitHub Pages via GitHub Actions.

---

## The n8n Idea

Early in the planning, I considered using **n8n** (a self-hosted workflow automation tool) as an **AI agent orchestrator** — not to replace GitHub Actions, but to sit on top of it. The idea: n8n listens to GitHub webhooks, triggers an AI agent on each commit, and has the agent do things like auto-update `intro.md`, generate summaries, or update a changelog.

- Visual workflow editor to wire up the agent logic
- Could generalize to other org automations later
- Appealing as a "smart layer" above the CI pipeline

### Why I didn't go with it

1. **It needs hosting.** n8n requires a running server — a VM, container, or k3s pod. That's infrastructure to maintain before even having a stable cluster.

2. **The agent problem is the wrong problem to solve.** The reason `intro.md` would go stale isn't a lack of automation — it's that it was designed to track content. Rewriting it to be permanently static eliminated the need for an agent entirely.

3. **AI-generated commits in a learning repo.** Messy git history, silent failures, and you lose visibility into what changed and why.

**Decision: n8n/agent shelved. Revisit once the org has a stable cluster and a real cross-system automation need.**
